Проект TODO-list с REST API и MySQL будет работать так:

Есть база данных MySQL, в которой таблица tasks хранит задачи (с полями id, title, status и т.д.).

Бэкенд на Python (например, FastAPI или Flask) подключается к базе через библиотеку, например, SQLAlchemy или mysql-connector.

В бэкенде выполняются SQL-запросы (через ORM или напрямую) для операций CRUD (создать, прочитать, обновить, удалить задачи).

Пример метода: при GET-запросе к /tasks бэкенд через SQLAlchemy выполняет session.query(Task).all(), получает список задач.

Результат запроса (например, список задач) сериализуется в JSON и возвращается клиенту как HTTP-ответ.

На фронтенде — простой HTML с JavaScript, который с помощью fetch() отправляет запросы к API (например, GET /tasks).

Полученные JSON-данные загружаются в HTML-блок (например, <ul>), где через DOM-операции рисуется список задач.

При добавлении/обновлении задачи фронтенд отправляет POST/PUT-запрос с JSON-данными на бэкенд, тот обновляет базу.

После успешного ответа фронтенд обновляет отображение страницы динамически через JavaScript, без перезагрузки.

Для запуска проекта используется виртуальное окружение Python, где установлены FastAPI, SQLAlchemy и mysql-connector.

Итог:

MySQL — хранит данные.

Python (с ORM) — управляет данными, предоставляет REST API.

HTML+JS — клиент, взаимодействует с API через HTTP-запросы, обновляет страницу динамически.
